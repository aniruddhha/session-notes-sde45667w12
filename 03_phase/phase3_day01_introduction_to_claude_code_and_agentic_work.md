# Phase 3 -> Day 1
## Introduction to Claude, Claude Code, and General-Purpose Agentic Work
### Trainer Session Notes

**Session Duration:** 1 hour 30 minutes  
**Session Mode:** 20% Theory | 80% Hands-On Discussion / Demonstration  
**Primary Environment:** Antigravity  
**Primary Tool Focus:** Claude Code  
**Important Framing:** Claude Code is not taught only as a coding tool. It is taught as a general-purpose agentic work assistant.

---

## 1. Session Goal for the Trainer

By the end of this session, learners should understand:

- what Claude is
- what Claude Code is
- why Claude Code should be understood as an agentic work assistant
- how Claude Code differs from normal chatbot usage
- why Antigravity is useful as an agent workspace
- how agentic work applies to documents, planning, review, research, automation, testing, content work, and coding
- why human control, review, permissions, and instructions are important
- what will be covered across Phase 3

**Trainer objective:**  
This session should open the learner’s mind. They should not think:  
“Claude Code is only for programmers.”

They should think:  
“Claude Code is an agentic assistant that can work with files, context, tools, instructions, and workflows under human supervision.”

## 4. Core Theory: What Claude Is

Explain simply:

**Claude is an AI assistant created by Anthropic. It can understand natural language, reason over text, help draft content, analyze information, summarize documents, write code, review material, and assist with professional tasks.**

Do not over-focus on the model family.

Use this simplified explanation:

“Claude is the intelligence layer. Claude Code is one way of giving Claude access to a working environment where it can read files, understand context, use tools, and help complete tasks.”

### Trainer note
Keep the distinction clear:

- Claude = AI model / assistant
- Claude Code = agentic tool environment using Claude
- Antigravity = workspace where agentic work can be managed and reviewed

## 5. Core Theory: What Claude Code Is

Explain:

**Claude Code is an agentic tool that can work with a project, read files, understand instructions, use tools, run commands when permitted, create or edit files, and help complete multi-step tasks.**

Officially, Claude Code is described as an agentic coding tool that can read a codebase, edit files, run commands, and integrate with development tools. For this course, we will use that agentic capability more broadly for professional work.

### Important framing for learner
Say:

“Even though the name says Claude Code, the working idea is broader. If a task involves files, text, instructions, review, structure, planning, or automation support, Claude Code-style workflows can help.”

### Examples
- Review a folder and create an index
- Summarize multiple notes
- Create a project plan
- Draft documentation
- Generate a checklist
- Compare files
- Create a small script
- Review a policy draft
- Prepare testing scenarios
- Build a webpage
- Prepare a content plan

## 6. Chatbot Usage vs Agentic Usage

This is the most important distinction for Day 1.

### Chatbot-style usage
The user asks a question and receives an answer.

Examples:
- “Explain AI in simple words.”
- “Rewrite this email.”
- “Summarize this paragraph.”
- “Give me five ideas.”

### Agentic usage
The user gives a goal. The assistant may inspect context, plan steps, use tools, work with files, create artifacts, and continue until the task is completed or reviewed.

Examples:
- “Review this folder and create an index before and after restructuring.”
- “Read these notes and create a training plan.”
- “Analyze this project and find missing documentation.”
- “Create a test plan from this requirement file.”
- “Build a simple webpage and verify it in the browser.”
- “Check this document for privacy, bias, and hallucination risk.”

### Trainer explanation
Say:

“In chatbot usage, the output is usually the answer. In agentic usage, the output may be a file, a plan, a checklist, a modified project, a report, a test result, or a reviewed artifact.”

## 7. General-Purpose Agentic Work

Explain:

**General-purpose agentic work means using an AI assistant to complete multi-step professional work by combining instructions, context, files, tools, and human review.**

This can include:

- writing
- reviewing
- planning
- checking
- organizing
- comparing
- documenting
- testing
- researching
- creating tools
- analyzing workflows
- preparing domain-specific outputs

### Core idea
Claude Code should be treated as:

**an assistant that can work inside a work environment, not just a chat window.**

### Verbatim explanation
“Agentic work begins when AI is not only answering, but helping perform a task step by step.”

## 9. Role of Antigravity

Explain:

**Antigravity is the environment where agentic work can be managed visually and practically.**

It can be introduced as an agent workspace where the assistant can operate across:

- editor
- files
- terminal
- browser
- artifacts
- task plans
- screenshots or recordings where supported
- review outputs
- agent manager

### Trainer explanation
Say:

“Antigravity helps us see the work. It is not only where files are edited. It is where the agent’s plan, output, verification, and artifacts can be reviewed.”

### Simple comparison

| Normal Chat | Antigravity Agent Workspace |
|---|---|
| Ask question, get answer | Give task, review plan and outputs |
| Mostly text-based | Files, editor, terminal, browser, artifacts |
| Limited workflow visibility | Better task visibility |
| User manually does all execution | Agent can assist execution with permission |
| Output is mostly answer | Output can be files, diffs, plans, screenshots, reports |

## 10. Human Control and Safety

This must be introduced from Day 1.

Claude Code and Antigravity workflows are powerful because they may involve tools and files. Therefore learners must understand:

- AI should not be given uncontrolled authority
- humans define the task
- humans approve risky actions
- humans review outputs
- sensitive data must be protected
- agent actions should be visible
- generated outputs must be validated
- permissions matter
- final responsibility remains human

### Verbatim explanation
“An agentic assistant is useful because it can act. But because it can act, we must control what it can access, what it can modify, and what must be reviewed before use.”

## 11. Key Terms to Introduce on Day 1

Do not teach these deeply yet. Just introduce them.

### Claude
The AI assistant / model.

### Claude Code
The agentic working tool that allows Claude to work with files, tools, commands, and project context.

### Antigravity
The agent workspace used to manage tasks, files, browser, terminal, plans, and artifacts.

### Agent
An AI assistant that can take a goal, plan steps, use tools, and work toward completion.

### Context
The information available to the AI, such as prompts, files, instructions, previous outputs, and project rules.

### Tool
A capability the agent can use, such as reading files, editing files, running terminal commands, or interacting with a browser.

### Permission
Human control over what actions the agent can perform.

### Artifact
A reviewable output created during agent work, such as a plan, file, diff, screenshot, diagram, or report.

### Memory
Stored preferences or project context that helps future work.

### Skill
A reusable capability or task pattern.

### Subagent
A specialized assistant for a specific role or part of a larger task.

### MCP
A way to connect AI agents to external tools, data, and systems.

## 12. Demo Discussion 1: Same Task in Chatbot vs Agentic Mode

### Task
“Create a project plan from rough notes.”

### Chatbot mode
User pastes rough notes.  
AI gives a plan in the chat.

### Agentic mode
AI can:
1. read notes from files
2. inspect folder structure
3. create a project plan file
4. create a checklist
5. ask for confirmation
6. update the document
7. create final summary

### Trainer point
Say:

“The difference is not only answer quality. The difference is workflow capability.”

## 13. Demo Discussion 2: General Work Example

### Scenario
A folder contains:
- meeting notes
- rough requirements
- screenshots
- draft document
- old checklist

### Agentic task
“Review this folder and create a structured project briefing with missing information, risks, and next actions.”

### Claude Code-style workflow
1. Read files
2. Identify document types
3. Summarize each file
4. Extract common themes
5. Identify missing information
6. Create briefing note
7. Create next-action checklist
8. Ask human to review

### Trainer point

## 14. Demo Discussion 3: Coding as One Use Case

### Scenario
The user needs a Python script to reorganize files.

### Agentic workflow
1. Understand folder rules
2. Ask risk questions if needed
3. Create dry-run plan
4. Create script
5. Generate before index
6. Move/rename files only after approval
7. Generate after index
8. Create report

### Trainer point
This includes coding, but the real workflow includes planning, safety, validation, and documentation.

6. Create a simple website from a written requirement.

8. Summarize a public document.

## 18. Common Misunderstandings to Correct

### Misunderstanding 1
“Claude Code is only for software developers.”

Correction:
“Claude Code is strongest in code-related environments, but the agentic concepts apply to many professional workflows involving files, instructions, tools, review, and outputs.”

### Misunderstanding 2
“Agent means AI should do everything automatically.”

Correction:
“Agent means AI can work through steps, but humans define boundaries, permissions, and approval.”

### Misunderstanding 3
“Antigravity is just another IDE.”

Correction:
“Antigravity should be understood as an agent workspace where tasks, artifacts, files, terminal, and browser can be used together.”

### Misunderstanding 4
“If AI can complete the task, human review is optional.”

Correction:
“Human review becomes more important when AI can act on files and tools.”
