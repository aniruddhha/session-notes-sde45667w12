# Phase 2 -> Day 3  
## Structured Prompting for Professional Work  
### Trainer Session Notes

**Session Duration:** 1 hour 30 minutes  
**Session Mode:** 20% Theory | 80% Hands-On  
**Primary Tools for Today:** ChatGPT, Claude, Gemini, DeepSeek, Grok  
**Phase Focus:** Safe, repeatable, professional prompting workflows

---

## 1. Session Goal for the Trainer

By the end of this session, participants should:
- understand the difference between casual prompting and structured prompting
- learn how to build reusable prompt templates for common professional tasks
- practice structured prompting for summarization, drafting, review, analysis, comparison, tabulation, and action planning
- understand how structured prompts improve consistency, reviewability, and safety
- learn to include boundaries such as “do not add facts,” “mark assumptions,” and “list verification needs”
- begin thinking of prompts as repeatable work instructions, not one-time questions

**Your teaching objective:**  
Today is about moving from one-off prompting to repeatable professional prompting.  
Do not teach this as a list of random prompts.  
Teach it as a disciplined method for converting recurring work into reusable AI-assisted workflows.  
The participant should feel:  
“Now I can create reusable prompt templates for common professional work and adapt them safely.”

## 4. Core Theory You Must Cover Briefly

Keep this section practical and template-focused.

### 4.1 What structured prompting means
Use this explanation:

**Structured prompting means giving AI a clear, reusable instruction pattern that includes task definition, input context, output format, constraints, and review expectations.**

Then simplify:

**Structured prompting turns a prompt into a repeatable work template.**

### 4.2 Difference between casual and structured prompts

Casual prompt:
“Summarize this.”

Structured prompt:
“Act as a professional briefing assistant. Summarize the following text into five bullet points. Preserve all decision-critical details. Do not add new facts. At the end, list missing information and verification needs.”

### 4.3 Why structured prompting matters
Explain:
- improves consistency
- reduces guesswork
- makes outputs easier to review
- supports repeatable workflows
- helps avoid missing key fields
- makes assumptions visible
- reduces the risk of unsupported additions
- helps maintain tone and format

### 4.4 Where structured prompting is most useful
Clearly name these areas:
- summarization
- drafting
- rewriting
- review and critique
- comparison
- tabulation
- action planning
- risk identification
- decision-preparation support
- meeting notes
- briefing notes
- checklist creation

### 4.5 Key structure
Use this repeatable frame:

1. **Role** — Who should AI act as?
2. **Input** — What information is provided?
3. **Task** — What should AI do?
4. **Output format** — How should the result look?
5. **Rules** — What must AI include or avoid?
6. **Review section** — What should AI flag?

### 4.6 Professional safety addition
Say this clearly:

**Every serious professional prompt should make assumptions and verification needs visible.**

### 6.1 The simplest mental model
You should say:

“Think of structured prompting like creating a standard operating instruction for AI. Instead of asking casually every time, we define how the task should be done, what format the output should follow, and what risks should be flagged.”

### 6.2 Another simple explanation
You may say:

“If normal prompting is like giving a quick instruction, structured prompting is like giving a professional format with clear fields, rules, and review checkpoints.”

### 6.3 Important caution
Also say:

“Structured prompts improve consistency, but they do not eliminate the need for human review. A well-formatted wrong answer is still a wrong answer.”

### 6.4 Important maturity point
Say this clearly:

“The mature user builds prompt templates for recurring work, tests them, improves them, and keeps review built into the workflow.”

## 8. Demo 1 — Structured Summarization Prompt

### Purpose
To show how a basic summary becomes professional and reviewable with structure.

### Rough input
“meeting happened monday. documents incomplete. one report pending. follow up next monday. decision should wait. communication needed but no blame.”

### Casual prompt
“Summarize this.”

### Structured prompt
“Act as a professional briefing assistant. Summarize the following rough notes into a concise briefing with sections: context, key observations, pending actions, risks, and next step. Preserve decision-critical details. Do not add facts not provided. Use neutral language. At the end, list missing information and verification needs.”

Run in:
- ChatGPT
- Claude
- Gemini
- DeepSeek
- Grok

### What participants should observe
- structured prompt produces better headings
- important conditions are more likely retained
- missing information becomes visible
- output is easier to review
- tone is controlled

### Verbatim you can say
“This is the power of structured prompting. We are not just asking for a summary. We are defining what kind of summary we need, what sections it should contain, what tone it should use, and what risks should be visible.”

### Key insight to emphasize
**Structured summarization preserves usefulness and improves reviewability.**

## 9. Demo 2 — Structured Drafting and Rewriting Prompt

### Purpose
To show how structured prompting improves professional communication.

### Input
Use the briefing generated in Demo 1.

### Structured drafting prompt
“Act as a professional communication assistant. Convert the following briefing into a short internal email. Keep the tone neutral, respectful, and non-accusatory. Do not add new facts. Include: purpose, current status, pending action, and next step. Keep it under 150 words. At the end, mention if any information is missing before sending.”

### Follow-up prompt
“Now rewrite the same email for senior leadership in a more concise version under 80 words, while preserving all critical facts.”

### What participants should observe
- same input can produce different audience-specific outputs
- structure controls tone and length
- follow-up adapts output without changing facts
- “do not add new facts” is important
- missing information section supports review

### Verbatim you can say
“Professional drafting is not only about sounding formal. It is about preserving facts, controlling tone, matching the audience, and making sure nothing unsupported is added.”

### Key insight to emphasize
**Structured drafting helps create audience-fit communication without losing control.**

---

## 10. Demo 3 — Structured Comparison and Table Prompt

### Purpose
To show how structured prompts help compare options clearly.

### Scenario
Compare ChatGPT, Claude, Gemini, DeepSeek, and Grok for a professional task.

### Structured comparison prompt
“Compare ChatGPT, Claude, Gemini, DeepSeek, and Grok for the task of summarizing rough professional notes. Use a table with columns: tool, likely strength, possible limitation, best use case, and review caution. Keep the comparison practical. Do not claim one tool is always best. Mention that outputs should be tested for the actual task.”

### What participants should observe
- structured table improves clarity
- comparison becomes easier to scan
- caution is built into the output
- tool comparison is task-based, not brand-based
- review discipline remains included

### Verbatim you can say
“When we compare tools, the goal is not to declare a permanent winner. The goal is to understand which tool may fit which task, and what review caution remains.”

### Key insight to emphasize
**Structured comparison makes decision-support output easier to inspect.**

---

## 11. Demo 4 — Structured Review and Critique Prompt

### Purpose
To show that AI can help review AI-generated or human-generated text.

### Prompt
“Review the following draft for accuracy, completeness, tone, unsupported assumptions, missing information, and risk. Present your review in a table with columns: issue found, why it matters, suggested correction, and verification needed. Do not rewrite the full draft unless asked.”

### What participants should observe
- AI can support critique
- review prompt makes issues visible
- table improves actionability
- human still decides final corrections
- critique is useful for quality control

### Verbatim you can say
“This is a valuable pattern. We can use AI not only to generate content, but also to inspect content. However, AI review is still support. It does not replace human review.”

### Key insight to emphasize
**Structured review prompts turn AI into a quality-checking assistant.**

## 12. Core Professional Prompt Templates You Should Teach

### 12.1 Summarization template
“Act as a professional briefing assistant. Summarize the following content into [number] key points. Preserve all decision-critical details. Do not add facts not present in the source. Use [tone] tone. End with missing information and verification needs.”

### 12.2 Drafting template
“Act as a professional communication assistant. Draft a [email/note/report/briefing] for [audience] based only on the following points. Include [sections]. Keep it [length]. Use [tone]. Do not add unsupported facts. Mention assumptions separately.”

### 12.3 Rewriting template
“Rewrite the following text for [audience/purpose]. Improve clarity, tone, and structure without changing meaning. Keep [formal/neutral/simple/concise] tone. Do not add new information. Mention any unclear points.”

### 12.4 Extraction template
“Extract information from the following text into a table with columns [columns]. If a field is missing, write ‘not provided.’ Do not infer missing data. Add a final section listing verification needs.”

### 12.5 Comparison template
“Compare the following options using criteria [criteria]. Present the result in a table. Mention strengths, limitations, and best-fit use cases. Do not declare a final recommendation unless clearly supported by the provided context.”

### 12.6 Review template
“Review the following output for accuracy, completeness, relevance, tone, unsupported assumptions, and risk. Present issues in a table with suggested corrections and verification needs.”

### 12.7 Action-plan template
“Create a preliminary action plan from the following information. Separate immediate actions, pending information, responsible party if mentioned, timeline if mentioned, and verification required. Do not assume missing details.”

## 13. Activity — Build Reusable Prompt Templates

### Duration
12 minutes

### Objective
Participants create their own reusable prompts.

### Instructions
Ask participants to build one prompt template for any two of these:
- summarizing a long document
- drafting a formal email
- extracting action items
- comparing options
- reviewing an AI-generated output
- creating an action plan

### Required template fields
Each prompt must include:
- role
- task
- input context
- output format
- constraints
- verification or assumptions section

### Verbatim you can say
“Your goal is not to create the longest prompt. Your goal is to create a reusable prompt that gives controlled, structured, reviewable output.”

### Trainer note
After they create prompts, ask:
- Can this be reused?
- What should be changed for another task?
- Does it prevent unsupported assumptions?
- Does it define output format?
- Does it include a review section?

## 14. High-Value Concepts to Emphasize During Discussion

### 14.1 Reusable prompts create consistency
Explain:
- repeated work benefits from repeated structure
- teams can standardize summaries and briefings
- review becomes easier when formats are consistent

### 14.2 Templates reduce mental load
Explain:
- users do not need to start from zero
- common workflows become faster
- quality improves over repeated use

### 14.3 Structure improves reviewability
Explain:
- tables expose missing fields
- headings preserve important categories
- assumptions section makes uncertainty visible

### 14.4 Templates must be adapted
Explain:
- same template cannot fit every situation unchanged
- audience, risk, sensitivity, and desired format matter
- users must adjust before use

### 14.5 Structured prompts support safe usage
Repeat:
- do not add facts
- mark assumptions
- list verification needs
- do not make final decisions
- use neutral tone

- What should the model avoid?

- What verification needs should be listed?

- Can this template be reused safely?

## 17. Risks and Cautions to Mention Even in This Session

Mention these calmly:
- templates can create false confidence if not reviewed
- wrong templates can repeat wrong structure
- reusable prompts must still be adapted to context
- “do not add facts” reduces but does not eliminate hallucination risk
- structured output may look reliable even when content is weak
- sensitive data should still be minimized

Use this line:

**A reusable prompt is useful only when it is also reviewable and safe.**
