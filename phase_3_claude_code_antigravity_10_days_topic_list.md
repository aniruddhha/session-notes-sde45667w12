# Phase 3: Claude Code as a General-Purpose Agentic Work Tool

## Phase Duration
**10 Days**

## Phase Purpose

This phase introduces Claude Code as a **general-purpose agentic work assistant**, not only as a coding tool.

The goal is to help learners understand how Claude Code can be used for:

- document creation and review
- planning and task breakdown
- research-style analysis
- file and folder understanding
- workflow support
- automation assistance
- writing and rewriting
- checklist creation
- testing and review support
- coding and codebase work
- domain-specific productivity tasks

Antigravity will be used as the working environment to understand how an AI agent can operate across files, editor, terminal, browser, artifacts, and multi-step workflows.

This phase is mainly **theory + conceptual understanding**.  
The next phase will focus on **practical demonstrations using Claude Code and Antigravity**.

## Day 2: Claude Code Mental Model: Agent Loop, Tools, Context, Files, and Human Control

### Core Idea
Learners should understand how Claude Code thinks and works at a high level.

### Topics to Cover
- What an agent loop is
- How Claude Code receives a task
- How it reads context
- How it plans
- How it uses tools
- How it edits or creates files
- How it asks for permission
- How the human reviews output
- Difference between one-step task and multi-step task
- How Claude Code uses project files as context
- Why context quality affects output quality
- Why human control is necessary

### Agent Loop Concept
A simple loop to explain:

1. Understand the task
2. Read available context
3. Plan possible steps
4. Use tools if allowed
5. Produce or modify output
6. Ask for confirmation when needed
7. Human reviews and approves
8. Continue or stop

### Examples to Discuss
- “Review this folder and create an index”
- “Create a project plan from these notes”
- “Find gaps in this document”
- “Generate a checklist from this policy”
- “Explain this codebase”
- “Create a small script after reviewing requirements”

### Trainer Emphasis
Claude Code is powerful because it can combine **context + tools + files + instructions**, but it must remain under **human direction and review**.

## Day 10: Designing General AI-Agent Workflows: Documentation, Planning, Review, Research, Coding, Testing, and Domain Tasks

### Core Idea
This day consolidates Phase 3 theory into a complete understanding of how Claude Code can be used as a general-purpose professional agent.

### Topics to Cover
- What makes a good AI-agent workflow
- Choosing the right task for Claude Code
- Defining task boundaries
- Preparing project instructions
- Using memory safely
- Using skills for repeatable work
- Using tools with permission
- Using subagents for multi-step work
- Using MCP for external context
- Using Antigravity for visibility and review
- Human approval and accountability
- Designing reusable workflow templates

### General Workflow Pattern
1. Define work objective
2. Classify risk
3. Prepare files and context
4. Write or update `CLAUDE.md`
5. Decide whether memory is needed
6. Choose relevant skill
7. Define allowed tools
8. Ask Claude Code to plan
9. Review plan
10. Execute step-by-step
11. Validate output
12. Human approves final use

### Example Workflow Areas
- document review workflow
- research summary workflow
- policy checklist workflow
- software testing workflow
- website creation workflow
- security checklist workflow
- video content planning workflow
- government note preparation workflow
- file restructuring workflow
- codebase understanding workflow

### Trainer Emphasis
Day 10 should make learners confident that Claude Code is not just a coding assistant. It is a **general-purpose agentic work platform** when used with proper instructions, memory, skills, tools, permissions, and human oversight.
